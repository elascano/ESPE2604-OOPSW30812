Description:
A robust Java-based application designed to demonstrate the effective
management and organization of data structures. This project strictly
follows the MVC (Model-View-Controller) architectural pattern to
ensure a clean separation of concerns, readability, and high
maintainability.

📂 Project Structure
The source code is organized into the following main packages and
classes:

1. Packages:
   - `model`: Contains the core data representation and logic.
     Includes classes defining the data entities managed by the
     application.
   - `view`: Manages the user interface and is responsible for
     how information is presented to the user.
   - `controller`: Acts as the essential bridge between the Model
     and the View, handling user input, updating the data, and
     invoking the correct view.

2. Key Components:
   - `Person` Class: Found within the `model` package, this is the
     fundamental data entity representing an individual with specific
     attributes.
   - `CollectionsApp`: The primary executable entry point of the
     application. This class initializes the MVC system and coordinates
     the flow between all components.
